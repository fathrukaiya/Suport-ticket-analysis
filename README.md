# Support Ticket Analysis

## 📌 Overview
This project analyzes support ticket data to identify trends, recurring issue categories, SLA risks, backlog growth, and overall service performance.

The objective is to help support teams improve response time, reduce repeated incidents, and make data-driven operational decisions.

---

## 🛠 Tools Used
- SQL
- Excel / CSV
- Power BI
- GitHub

---

## 📂 Project Structure
- `sql/queries.sql` → SQL analysis queries
- `data/tickets.csv` → support ticket dataset
- `dashboard/powerbi_screenshots.png` → dashboard screenshots
- `README.md` → documentation

---

## 🎯 Business Questions Solved
- What are the most common ticket categories?
- Which tickets are still unresolved?
- What is the ticket volume trend by day?
- Which priorities remain open the longest?
- Which agents close the most tickets?

---

## 📈 Key Insights
- Identified top recurring issue categories
- Measured unresolved ticket backlog
- Highlighted high-priority pending tickets
- Tracked daily support demand
- Improved visibility into support operations

---

## 🚀 Outcome
This project demonstrates how SQL and dashboarding convert raw support data into actionable business insights for operational efficiency.
